# Entrevista con Stakeholders para el Sistema de Gestión de Continuidad en Cine y TV

## Problemas Actuales y Oportunidades (15 minutos)
1. Actualmente, ¿cuáles son los mayores desafíos o "dolores de cabeza" que experimentas en relación con la continuidad en las producciones? (Ej. errores, inconsistencias, pérdida de información, comunicación, etc.)
2. ¿Podrías describir alguna situación específica en la que una inconsistencia o un error de continuidad haya causado problemas significativos (regrabaciones, costos adicionales, retrasos)?
3. En cuanto a la información confidencial (guiones, fotos de set, etc.), ¿cómo se maneja actualmente? ¿Existen preocupaciones sobre la seguridad o el control de esta información?
4. ¿Qué herramientas o métodos utilizas actualmente para gestionar la continuidad y la comunicación en el set (mensajería, hojas de cálculo, notas manuales, etc.)? ¿Qué te gusta o no te gusta de estas herramientas?
5. Si pudieras cambiar algo del proceso actual de gestión de continuidad, ¿qué sería lo primero que harías?
## Requisitos Funcionales (RFs) - Lo que el sistema debe hacer (20 minutos)
6. ¿Qué tipo de material fotográfico es crucial almacenar para cada escena? (Ej. fotos de vestuario, peinado, maquillaje, utilería, set, ángulos de cámara, etc.)
7. ¿Cómo se debería etiquetar o categorizar este material fotográfico para que sea fácil de encontrar? ¿Qué detalles son importantes registrar (personaje, escena, toma, fecha, descripción, comentarios)?
8. ¿Sería útil un sistema que analizara automáticamente las imágenes para identificar elementos clave (ej. vestuario específico, accesorios, peinado)? ¿Qué tan preciso esperas que sea?
9. ¿Necesitas poder comparar imágenes de diferentes tomas o escenas para asegurar la consistencia? 
10. ¿Cómo te gustaría que se presentara esa comparación?
11. ¿Quién debería tener acceso para subir, editar o eliminar fotos y descripciones?
Módulo de Planeación y Comunicación:
12. ¿Cómo se organiza actualmente el calendario de rodaje y los desgloses?
13. ¿Qué tipo de información debería incluir un calendario centralizado del rodaje? (Ej. escenas a grabar, personajes involucrados, locaciones, equipo específico, etc.)
14. ¿Qué tan importante es recibir notificaciones instantáneas sobre cambios en el rodaje? ¿A quiénes deberían llegar estas notificaciones y por qué medios (ej. móvil, email, dentro de la app)?
15. ¿Quién debería poder crear o modificar eventos en el calendario? ¿Hay diferentes niveles de acceso para ver la información del calendario?
16. ¿Necesitas una forma de comunicar mensajes generales al equipo o mensajes específicos a grupos/individuos? ¿Qué funcionalidades de comunicación son esenciales?
17. ¿Quiénes necesitan tener acceso al sistema y qué tipo de información deberían poder ver o modificar? (Ej. director, continuidad, vestuario, maquillaje, producción, actores, etc.)
18. ¿Qué tan importante es que la información (guiones, fotos, planes de rodaje) esté protegida contra descargas no autorizadas o filtraciones?
19. ¿Se necesitan diferentes niveles de permisos para distintas secciones del sistema? ¿Podrías dar ejemplos?
20. ¿Qué preocupación tienes sobre el acceso a la información una vez que una persona deja el proyecto?
21. ¿Qué tipo de informes o resúmenes te gustaría poder generar con el sistema? (Ej. listado de vestuario por personaje, resumen de cambios en el guion, etc.)
22. ¿Hay alguna otra funcionalidad que consideres esencial para el sistema, algo que no hayamos mencionado?
## Requisitos No Funcionales (RNFs) - Cómo debe funcionar el sistema (15 minutos)
23. ¿Cuántos usuarios simultáneos esperarías que usen el sistema en un momento dado, especialmente en el set?
24. ¿Qué tan rápido esperas que el sistema cargue imágenes, guiones o información del calendario? ¿Hay algún retraso inaceptable?
25. ¿El sistema debe funcionar sin conexión a internet en el set o en locaciones remotas? Si es así, ¿cómo debería sincronizarse la información una vez que haya conexión?
26. ¿Qué tan fácil de aprender y usar debería ser el sistema para personas con diferentes niveles de habilidad tecnológica?
27. ¿Hay algún dispositivo específico (tablets, móviles, computadoras de escritorio) en el que el sistema deba funcionar bien?
28. ¿Es importante que la interfaz sea visualmente intuitiva y fácil de navegar, especialmente en un entorno de set ajetreado?
29. ¿Qué tan crítico es que el sistema esté disponible 24/7? ¿Qué impacto tendría una caída del sistema durante el rodaje?
30. ¿Con qué frecuencia se debería hacer una copia de seguridad de la información?
31. ¿Qué tan rápido se debería poder restaurar la información en caso de pérdida?
32. ¿El sistema necesita soportar proyectos de diferentes tamaños (cortometrajes, series, películas grandes)?
33. ¿Se espera que el número de usuarios o la cantidad de datos crezca significativamente con el tiempo?
34. ¿Qué medidas de seguridad adicionales crees que son importantes (ej. cifrado de datos, autenticación de dos factores, historial de accesos)?
35. ¿Qué tan importante es proteger la información contra ataques externos?
Mantenibilidad:
36. ¿Se espera que el sistema se actualice o evolucione con nuevas funcionalidades en el futuro?

37. ¿Existen presupuestos o plazos específicos que debamos tener en cuenta para el desarrollo de este sistema?
38. ¿Hay alguna tecnología o plataforma específica que deba usarse o evitarse? (Ej. "debe ser web", "no puede ser una aplicación descargable", "debe integrarse con [X software]").
39. ¿Existen políticas internas de la empresa o regulaciones de la industria que el sistema deba cumplir?
40. ¿Hay alguna otra limitación o factor restrictivo que debamos considerar?
